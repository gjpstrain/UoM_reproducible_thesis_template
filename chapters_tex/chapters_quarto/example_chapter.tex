\section{Introduction}\label{introduction}

Welcome to the unofficial University of Manchester reproducible thesis
template. My PhD project was done in an entirely open and reproducible
manner, including all papers and my final thesis. In writing my
\href{link_here}{thesis}, I developed this template in case anyone else
wanted to have a go at writing a fully reproducible thesis. I am an R
user exclusively, so I wrote my thesis using
\href{https://quarto.org/}{quarto} and the
\href{https://posit.co/download/rstudio-desktop/}{Rstudio} Integrated
Development Environment, although everything here also works with {[}VS
Code{]})(https://code.visualstudio.com/). In this example chapter, I
cover:

\begin{itemize}
\tightlist
\item
  The general workflow for using the template.
\item
  How to write using quarto-flavoured markdown.
\item
  How to use citations, cross-referencing, and labels.
\item
  How to use code chunks to generate figures and tables, including how
  to use pre-rendered graphics.
\item
  How to use code chunks for data wrangling and statistical modelling.
\item
  How to dynamically include the outputs of those code chunks in-text.
\item
  How to use the scripts provided to render your final document.
\end{itemize}

\subsection{A Note for Users of Other
Languages}\label{a-note-for-users-of-other-languages}

Quarto uses markdown syntax. I use Rstudio as my IDE. \texttt{knitr} is
the engine that interprets code chunks when using Rstudio, and it
supports a variety of languages, including R, Python, SQL, JavaScript,
and Ruby. For a full list of supported languages, type the following
command:

\texttt{names(knitr::knit\_engines\$get())}

\section{Workflow and Writing}\label{workflow-and-writing}

\subsection{The Workflow}\label{the-workflow}

This template has been set up to make it as easy as possible to get
started writing your fully-reproducible thesis. The general workflow is
provided below:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Chapters are stored as \texttt{.qmd} files in the
  \texttt{chapters\_quarto} folder in the parent directory. You should
  write each chapter separately using these files.
\item
  Once you are happy with a draft of a chapter, add it to the
  \texttt{\%\ CHAPTERS\ \%} list in \texttt{main.tex}.
  \textbf{?@fig-chapters-tex} shows how to do this, and it must be done
  for every chapter.
\item
  Before producing a final thesis version, make sure to edit
  \texttt{main.tex} with your chapters, author information,
  declarations, abstracts, and acknowledgements.
\item
  Finally, use the \texttt{finalise\_thesis.R} script to render each
  chapter to a \texttt{.tex}, reformat the \texttt{.tex}, then knit them
  all together into the final thesis .pdf.
\end{enumerate}

\subsection{Writing With Quarto
Markdown}\label{writing-with-quarto-markdown}

\subsubsection{The Basics}\label{the-basics}

Quarto.org provides a great
\href{https://quarto.org/docs/authoring/markdown-basics.html}{guide} to
writing with quarto markdown. Ignore the image links, but everything
else works as documented. In brief:

\emph{italics}, \textbf{bold}, \textbf{\emph{bold italics}} are done
like this:

\texttt{*italics*,\ **bold**,\ ***bold\ italics***}

superscript\textsuperscript{2} / subscript\textsubscript{2} are done
like this:

\texttt{superscript\^{}2\^{}\ /\ subscript\textasciitilde{}2\textasciitilde{}}

\texttt{verbatim\ code} is done like this:

\texttt{verbatim\ code}

\subsubsection{Citations}\label{citations}

In this thesis template, numerical citations are used, with a single,
long bibliography inserted at the end. \texttt{references.bib} is kept
in the parent directory. In-text citations are done like this:
\texttt{...found\ in\ previous\ work\ @strain\_2023...}, producing this
text:

\ldots found in previous work @strain\_2023\ldots{}

\subsubsection{Cross-Referencing}\label{cross-referencing}

Chapters are referenced as such:

\texttt{As\ we\ can\ see\ in\ \textbackslash{}chap\{chap:example\_chapter\}...},
which produces the following:

As we can see in \chap{chap:example_chapter}\ldots{}

Sections are referenced similarly, but you need to type the word
``Section'' in addition.

\texttt{As\ in,\ in\ Section\ \textbackslash{}ref\{labelling\}...}
produces the following:

As in, in Section \ref{labelling}\ldots{}

You will probably end up with multiple sections called ``Introduction'',
``Methods'', ``Results'' etc. If this is the case, you can manually
specify the label for a section like so:

\begin{verbatim}
# Introduction_test {#introduction_chapter_1}

# Introduction_test {#introduction_chapter_2}
\end{verbatim}

Which allows you to do the following:

In Section X as opposed to Section Y.

Pages are referenced using \texttt{\textbackslash{}pageref\{\}}. This
does not automatically include the word ``page'', so you need to
manually add that in. You can point \texttt{pageref\{\}} to any chapter,
section, figure, or table like so:

\begin{verbatim}
\pageref{chap:example_chapter} # chapter

\pageref{introduction_chapter_1} # section

\pageref{fig-example} # figure

\pageref{tbl-example} # table
\end{verbatim}

You have two options for referencing figures and tables. If you are
referencing something in the same .qmd you are writing in, you can use
the \texttt{@} symbol like so:

\texttt{...as\ we\ can\ see\ in\ @fig-example\ and\ @tbl-example...},
which produces the following:

\ldots as we can see in Figure~\ref{fig-example} and
\textbf{?@tbl-example}\ldots{}

If you are referencing across different chapters, you need to use native
LaTeX referencing style (the same as we use for sections and pages).
When doing this, you need to manually add in ``Figure'' and ``Table'',
so writing this:

\texttt{...as\ we\ can\ see\ in\ Figure\ \textbackslash{}ref\{fig-example\}\ and\ Table\ \textbackslash{}ref\{tbl-example\}...}
produces the following:

\ldots as we can see in Figure \ref{fig-example} and Table
\ref{tbl-example}\ldots{}

\section{Figures \& Tables}\label{figures-tables}

For reproducibility, figures and tables should be generated from code
using code chunks. If you have pre-rendered graphics, such as
screenshots of experiments or figures that are taken from other
published work, you can use \texttt{knitr::include\_graphics()} to
include them, just as you would in native LaTeX. Below is a code chunk
that accomplishes this:

\begin{verbatim}

::: {.cell}
::: {.cell-output-display}
![Placeholder figure caption.](../supplied_graphics/chapters_tex_example.png){#fig-include-graphics width=NA height=NA}
:::
:::
\end{verbatim}

Code chunks can be inserted using a button in the top right corner of
this window, or by using \texttt{Alt+shift+I}. Have a look at
\texttt{example\_chapter.qmd} in \texttt{quarto\_chapters} for a demo of
this. Figure~\ref{fig-example} is produced using an r code chunk. All
code chunks are set to \texttt{echo:\ TRUE} in this example chapter so
that the pdf renders the chunk.

\begin{Shaded}
\begin{Highlighting}[]
\CommentTok{\# this is an r code chunk}
\CommentTok{\# generate some numbers, make a dataframe}

\FunctionTok{library}\NormalTok{(dplyr)}

\NormalTok{x }\OtherTok{\textless{}{-}} \FunctionTok{rep}\NormalTok{(}\DecValTok{1}\SpecialCharTok{:}\DecValTok{10}\NormalTok{)}
\NormalTok{y }\OtherTok{\textless{}{-}} \FunctionTok{runif}\NormalTok{(}\DecValTok{10}\NormalTok{)}

\NormalTok{df }\OtherTok{\textless{}{-}} \FunctionTok{tibble}\NormalTok{(x, y)}

\CommentTok{\# load plotting library, make basic plot}

\FunctionTok{library}\NormalTok{(ggplot2)}

\NormalTok{df }\SpecialCharTok{\%\textgreater{}\%}
  \FunctionTok{ggplot}\NormalTok{(}\FunctionTok{aes}\NormalTok{(}\AttributeTok{x =}\NormalTok{ x, }\AttributeTok{y =}\NormalTok{ y)) }\SpecialCharTok{+}
  \FunctionTok{geom\_smooth}\NormalTok{(}\AttributeTok{span =} \FloatTok{0.5}\NormalTok{, }\AttributeTok{se =} \ConstantTok{FALSE}\NormalTok{)}

\CommentTok{\# because we have \textasciigrave{}include\textasciigrave{} set to FALSE in the YAML header, output will not be automatically}
\CommentTok{\# included. Instead, we have to specify this in the code chunk (line 43 in the .qmd)}
\CommentTok{\# echo is set to TRUE in the chunk to display this chunk/code}
\end{Highlighting}
\end{Shaded}

\section{Labelling Figures}\label{labelling-figures}

\texttt{Figures\ are\ labelled\ as\ so:\ in\ @fig-example,\ we\ can\ see...}

Figures are labelled as so: in Figure~\ref{fig-example}, we can
see\ldots{}

If you are not generating figures from code, but rather including them
using knitr::include\_graphics(), you must set the out-height and
out-width to NA to prevent LaTeX from trying to manually size them.
